## Method Area
- Method Area는 가장 먼저 데이터가 저장되는 곳.
- 각 클래스 별로 고유한 정보를 저장
-Static(정적) 데이터이므로 프로그램이 종료될 때까지 메모리에 남아있음 => Method Area에서는 가비지 컬렉터(GC)가 동작 X
- Method Area 저장요소
    1) field data : 멤버 변수 이름, 타입, 접근제한자
    2) method data : 메서드 이름, 리턴타입, 매개변수, 접근제한자
    3) constructor(생성자) 메서드 정보
    4) static 변수(Class 변수) : 모든 객체가 공유할 수 있는 정적 클래스 변수
    5) runtime constant poo(런타임 상수 풀)
- 클래스 변수를 많이 사용하면 메모리 공간이 부족할 수 있다

## Heap Area
- 런타임(실행) 시 참조시 자료형이 저장
- new 연산자를 통해 생성된 객체(인스턴스)가 저장되는 공간
- 인스턴스 변수도 Heap 메모리 영역에 저장
- 객체가 더 이상 쓰이지 않거나, 명시적 null 선언 시 GC 가 삭제
- 모든 스레드가 공유

## Stack Area
- 컴파일 시 기본자료형 변수, 참조변수가 저장
- 메서드가 호출되면 메모리에 FILO로 삽입되고, 종료되면 LIFO로 제거
- 메서드가 호출될 때마다 각각의 스택 프레임 생성 => 각 스택 프레임은 하나의 메서드에 대한 정보를 저장
- 블록({ }) 혹은 메서드가 종료될 때 삭제 => 메서드 종료 시 프레임 별로 삭제
- 각 스레드 별로 생성

## PC 레지스터(PC register)
- JVM이 수행할 명령어의 주소를 저장하는 공간 => 운영체제의 레지스터와 유사한 역할이나 JVM이 별도로 관리
- 스레드가 시작될 때 마다 생성
- 각 스레드 별로 생성

## 네이티브 메서드 스택(Native Method Stack)
- 바이트 코드가 아닌 기계어로 작성된 코드를 실행하는 공간
- 다른 언어(C/C++ 등) 로 작성된 코드를 수행하기 위해 존재
- JNI(Java Native InterFace) 를 통해 바이트 코드로 변환
- 각 스레드 별로 생성
